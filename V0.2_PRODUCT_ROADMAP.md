## 2. 用户体验痛点分析

### 痛点矩阵

| 痛点 | 用户感知 | 当前状态 | 严重度 |
|------|----------|----------|--------|
| 无法暂停 | 战斗中无法休息，必须退出 | ❌ 无暂停功能 | **高** |
| 无法存档 | TMNT3 关卡长，死亡需从头开始 | ❌ 无 Save State | **高** |
| 无法全屏 | 小窗口体验差 | ❌ 仅 960x720 窗口 | **中** |
| 无法调音量 | 音量固定，深夜游戏扰民 | ❌ 音量不可调 | **中** |
| 键位不可改 | 不同玩家习惯不同 | ❌ 硬编码键位 | **中** |
| 无快速重试 | 死亡后需手动重启 ROM | ❌ 无 Quick Reset | **低** |
| 无最近 ROM 列表 | 每次需重新选择文件 | ❌ 无历史记录 | **低** |

### 用户旅程痛点

```
启动 -> 选择 ROM -> 开始游戏 -> 游戏中 -> 死亡 -> ???
  ✅      ✅        ✅        ❌暂停   ❌存档   ❌快速重试
```

## 4. 功能详情

### P0 — 配置系统

| 维度 | 内容 |
|------|------|
| 用户价值 | 后续所有可配置功能的基础（键位、窗口、音频）。无配置系统，每个功能都需硬编码默认值 |
| 技术复杂度 | 中 — 引入 `toml` 依赖，设计 `Config` 结构体，实现默认值回退 |
| 风险等级 | 低 — 新增依赖成熟，不影响现有功能 |
| 预计工作量 | 2-3 天 |
| tetanes-core 支持 | 不涉及，纯应用层 |

**实现要点：**
- 配置文件路径：`~/.config/turtlebox/config.toml`
- 无 config.toml 时使用硬编码默认值
- 配置结构：键位映射、窗口大小、音量默认值

---

### P1 — Pause（暂停/恢复）

| 维度 | 内容 |
|------|------|
| 用户价值 | TMNT3 战斗中可以休息，接听电话，查看攻略。当前无法暂停是最大体验缺陷 |
| 技术复杂度 | 低 — 主循环添加 `paused` 状态，暂停时跳过 `clock_frame()` |
| 风险等级 | 低 — 仅添加状态变量，不影响现有逻辑 |
| 预计工作量 | 0.5 天 |
| 推荐键位 | `P` 键（键盘） / `Start` 按钮（手柄） |

**实现要点：**
- 暂停时保持最后一帧渲染
- 暂停时显示 "PAUSED" 覆盖层（复用 `overlay::draw_text()`）
- 暂停时清空 `audio_buffer`，避免恢复时爆音

---

### P1 — Fullscreen（全屏切换）

| 维度 | 内容 |
|------|------|
| 用户价值 | 大屏体验提升明显，客厅游戏场景刚需 |
| 技术复杂度 | 低 — SDL2 `window.set_fullscreen()` 标准 API |
| 风险等级 | 低 — SDL2 全屏 API 成熟，macOS/Windows/Linux 均支持 |
| 预计工作量 | 0.5 天 |
| 推荐键位 | `F11` 或 `Alt+Enter` |

**实现要点：**
- 记住窗口化时的窗口位置和大小
- 可能遇到视网膜显示屏 DPI 问题（需测试）

---

### P1 — Volume（音量调节）

| 维度 | 内容 |
|------|------|
| 用户价值 | 深夜游戏不扰民，不同场景调节音量 |
| 技术复杂度 | 低 — 音频回调中乘以音量系数 |
| 风险等级 | 无 — 仅修改音频回调，不影响模拟逻辑 |
| 预计工作量 | 0.5 天 |
| 推荐键位 | `+` / `-`（键盘） / `LB` + `DPad Up/Down`（手柄） |

**实现要点：**
- `volume: f32` 状态（0.0 ~ 1.0，默认 1.0）
- 步进调节（每次 ±0.1）
- 音频回调：`out[i] = buffer[i] * volume`

---

### P1 — 自定义键位

| 维度 | 内容 |
|------|------|
| 用户价值 | 不同玩家习惯不同，左撇子、不同手柄布局等 |
| 技术复杂度 | 中 — 依赖配置系统，设计键位配置格式，运行时重新绑定 |
| 风险等级 | 低 — 配置文件驱动，不影响现有硬编码默认值 |
| 预计工作量 | 1-2 天（含配置系统集成） |

**配置格式示例：**
```toml
[input.keyboard.p2]
up = "Up"
down = "Down"
left = "Left"
right = "Right"
a = "A"
b = "D"
start = "Return"
select = "RShift"

[input.controller.p1]
a = "LeftShoulder"
b = "RightShoulder"
```

---

### P2 — Save State（快速存档）

| 维度 | 内容 |
|------|------|
| 用户价值 | TMNT3 关卡长，存档是刚需。Boss 战前存档，失败后快速重试 |
| 技术复杂度 | 中 — tetanes-core 已提供 `save_state(path)` / `load_state(path)` API |
| 风险等级 | 低 — tetanes-core 内部处理序列化，应用层仅调用 API |
| 预计工作量 | 1-2 天 |
| 推荐键位 | `F5` 存档 / `F8` 读档（标准模拟器惯例） |

**实现要点：**
- 存档槽位：1-9 个
- 存档路径：`~/.config/turtlebox/saves/{rom_name}/slot_{n}.sav`
- 存档时显示 "SAVED TO SLOT {n}" 提示
- 存档不存在时提示 "NO SAVE STATE"

---

### P2 — Quick Load（快速读档）

| 维度 | 内容 |
|------|------|
| 用户价值 | 快速恢复到上次存档点，死亡后秒恢复 |
| 技术复杂度 | 低 — 依赖 Save State 功能 |
| 风险等级 | 低 |
| 预计工作量 | 0.5 天（与 Save State 合并实现） |

---

### P2 — Quick Reset（快速重置）

| 维度 | 内容 |
|------|------|
| 用户价值 | 死亡后快速重新开始，无需退出重选 ROM |
| 技术复杂度 | 低 — 调用 `deck.unload_rom()` + `deck.load_rom_path()` 重新加载 |
| 风险等级 | 低 — 需测试重载后音频设备状态是否正常 |
| 预计工作量 | 0.5 天 |
| 推荐键位 | `Ctrl+R` 或 `F2` |

---

### P2 — Recent ROMs（最近 ROM 列表）

| 维度 | 内容 |
|------|------|
| 用户价值 | 启动时快速选择最近玩过的 ROM，无需每次浏览文件 |
| 技术复杂度 | 低 — 保存最近 10 个 ROM 路径到文件 |
| 风险等级 | 无 — 纯文件 I/O，不影响模拟逻辑 |
| 预计工作量 | 0.5 天 |

**实现要点：**
- 保存路径：`~/.config/turtlebox/recent.txt`
- 最多保存 10 个路径，LRU 策略
- 启动时验证路径是否仍然有效

## 5. 推荐开发顺序

### 原则

1. **先基础设施，后功能** — 配置系统是后续功能的基础
2. **先低风险高收益，后高风险** — 暂停、全屏、音量是低挂果实
3. **先用户刚需，后锦上添花** — Save State 优先于 Recent ROMs
4. **每个 Sprint 结束可编译运行** — 增量交付，随时可回退

### 推荐顺序

```
Sprint 2: 配置系统 (P0)
  ├── config.toml 解析 + 默认值回退
  ├── Config 结构体设计
  └── 配置文件路径管理

Sprint 3: 核心体验 (P1)
  ├── Pause（暂停/恢复）
  ├── Volume（音量调节）
  └── Fullscreen（全屏切换）

Sprint 4: 自定义键位 (P1)
  ├── 键位配置格式设计
  ├── 运行时键位读取
  └── 手柄按钮重映射

Sprint 5: 存档系统 (P2)
  ├── Save State（存档槽位）
  ├── Quick Load（快速读档）
  └── Quick Reset（快速重置）

Sprint 6: 体验优化 (P2)
  ├── Recent ROMs（最近 ROM 列表）
  ├── 错误处理改进（let _ = 替换）
  └── .gitignore 完善
```

## 6. Sprint 划分

### Sprint 2 — 配置系统（2-3 天）

**目标：** 建立配置基础设施，为后续功能提供配置支持

**任务：**
1. 添加 `toml` 依赖到 `Cargo.toml`
2. 创建 `src/config.rs` 模块
3. 设计 `Config` 结构体（键位、窗口、音频）
4. 实现 `config.toml` 解析 + 默认值回退
5. 配置文件路径：`~/.config/turtlebox/config.toml`
6. 集成到 `main.rs` 初始化流程

**验收标准：**
- `cargo build --release` 通过
- 无 config.toml 时使用硬编码默认值
- 有 config.toml 时正确读取配置
- 现有功能零变化

---

### Sprint 3 — 核心体验（1.5 天）

**目标：** 实现用户最直接感知的三个功能

**任务：**
1. Pause（0.5 天）— `P` 键 / `Start` 按钮切换暂停，显示 "PAUSED" 覆盖层，暂停时静音
2. Volume（0.5 天）— `+` / `-` 键调节音量，音频回调中应用音量系数
3. Fullscreen（0.5 天）— `F11` 键切换全屏，记住窗口化时的位置和大小

**验收标准：**
- `cargo build --release` 通过
- 暂停/恢复正常工作
- 音量可调节（0% ~ 100%）
- 全屏/窗口切换正常

---

### Sprint 4 — 自定义键位（1-2 天）

**目标：** 支持用户自定义键位映射

**任务：**
1. 设计键位配置格式（TOML）
2. 修改 `input.rs` 从 `Config` 读取键位映射
3. 支持键盘和手柄按钮重映射
4. 冲突检测（同一按键映射到多个功能）
5. 默认键位与当前硬编码一致

**验收标准：**
- `cargo build --release` 通过
- 无 config.toml 时使用默认键位（与 v0.1 一致）
- 有 config.toml 时使用自定义键位

---

### Sprint 5 — 存档系统（1.5-2 天）

**目标：** 实现快速存档/读档，解决 TMNT3 关卡长的痛点

**任务：**
1. Save State（1 天）— 调用 `deck.save_state(path)`，设计 1-9 槽位
2. Quick Load（0.5 天）— 调用 `deck.load_state(path)`，加载指定槽位
3. Quick Reset（0.5 天）— 调用 `deck.unload_rom()` + `deck.load_rom_path()` 重载

**验收标准：**
- `cargo build --release` 通过
- 存档/读档正常工作
- 存档槽位切换正常
- 快速重置后游戏重新开始

**推荐键位：**
- `F5` 存档 / `F8` 读档（标准模拟器惯例）
- `1`-`9` 切换存档槽位
- `Ctrl+R` 快速重置

---

### Sprint 6 — 体验优化（1 天）

**目标：** 完善细节，提升整体体验

**任务：**
1. Recent ROMs（0.5 天）— 保存最近 10 个 ROM 路径，启动时显示列表
2. 错误处理改进（0.3 天）— 替换 `let _ = canvas.fill_rect(...)` 为 `if let Err(e)`
3. `.gitignore` 完善（0.2 天）— 添加 `.DS_Store`、`*.nes`、`*.app/`、`*.sav`

**验收标准：**
- `cargo build --release` 通过
- 最近 ROM 列表正常工作
- 渲染错误不再静默丢弃

## 9. 验证方式

每个 Sprint 完成后必须验证：

1. `cargo build --release` — 编译通过
2. `cargo clippy` — 无新增 warning
3. 手动测试 — 运行 TMNT3 验证功能正常
4. `git commit` — 提交代码
5. `REFACTOR_SPRINTN_REPORT.md` — 生成报告

## 10. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| config.toml 解析失败 | 低 | 启动崩溃 | 默认值回退，解析失败时使用硬编码值 |
| 全屏切换后纹理拉伸 | 中 | 画面变形 | SDL2 自动处理，需测试不同分辨率 |
| Save State 文件损坏 | 低 | 读档失败 | `load_state` 返回错误时提示用户，保留多个槽位 |
| 手柄按钮冲突 | 中 | 功能异常 | 冲突检测，提示用户修改配置 |
| macOS 视网膜 DPI 问题 | 中 | 画面模糊 | 测试不同 Mac 机型，必要时手动处理 DPI |
