## 2. Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Escape` | Quit |
| `F1` | Toggle controls overlay |
| `F5` | Save state |
| `F8` | Load state |
| `F11` | Toggle fullscreen |
| `P` | Pause / resume |
| `=` | Volume +5% |
| `-` | Volume -5% |
| Arrow keys | P2 movement |
| `A` | P2 jump (NES A) |
| `D` | P2 attack (NES B) |
| `Enter` | P2 start |
| `Right Shift` | P2 select |

## 6. Known Issues

| Issue | Severity | Detail |
|-------|----------|--------|
| `block v0.1.6` future-compat warning | Low | Indirect dependency, pre-existing, not introduced by TurtleBox |
| Audio buffer growth | Low | No cap on audio buffer size; in practice NES output rate matches playback |
| Config path uses exe directory | Low | In .app bundles may write to unexpected location; works correctly for direct CLI |
| No runtime volume display | Low | Volume overlay shows % briefly; no persistent HUD |
| Controller only P1 | Low | Keyboard-only P2; no second controller support |

## 7. Risk Level

**Overall: LOW**

- All Sprint 4+5 features use well-tested tetanes-core APIs
- No unsafe code in TurtleBox
- No new dependencies added in Sprint 4/5
- Save state uses upstream serialization (bincode + deflate)
- All features gated by config with safe defaults
