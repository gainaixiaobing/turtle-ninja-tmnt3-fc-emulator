# TurtleBox v0.2 — Human Release Review

> 视角：第一次接触 TurtleBox 的用户，有 NES 模拟器使用经验。

## 1. First Run Experience

### 下载

用户拿到 TurtleBox.app 或 turtlebox 可执行文件。

**评估：** 假设通过某种渠道获得（GitHub release、朋友分享）。没有安装说明，但 macOS 用户通常知道双击 .app 或拖到 Applications。**无阻塞问题。**

### 启动

双击 TurtleBox.app。

**预期：** 弹出文件选择对话框，让我选 ROM 文件。

**实际流程：**
1. 双击 app
2. 文件对话框出现，过滤器显示 .nes 文件
3. 选择 TMNT3.nes
4. 游戏窗口出现，标题显示 "TurtleBox - TMNT3.nes"
5. 游戏开始运行

**评估：** 流程顺畅。文件对话框是标准系统组件，用户不需要学习。如果 TMNT3.nes 放在 app 旁边，甚至跳过对话框直接加载。**体验良好。**

### 加载 ROM

三种方式都清晰：
- 双击 → 对话框
- 拖拽 ROM 到 app
- 命令行指定路径

**评估：** 对普通用户来说，双击和拖拽是直觉操作。命令行方式对高级用户有用。**无问题。**

### 开始游戏

ROM 加载后游戏立即开始运行。

**评估：** 无等待、无配置、无教程。游戏直接进入。用户可以立刻开始玩。**优秀。**

## 3. Feature Discoverability

### Pause

**可发现性：低**

用户不会自然想到按 P 键暂停。大多数模拟器用 Start 暂停，但 TurtleBox 的 Start 按钮发送的是 NES Start（游戏内暂停），不是模拟器暂停。

**需要提示：** 在 F1 overlay 中已列出 P 键，但用户可能不会主动按 F1。

### Fullscreen

**可发现性：中**

F11 是 macOS 常见的全屏快捷键，有经验的用户会尝试。但普通用户可能不知道。

**需要提示：** F1 overlay 已覆盖。

### Volume

**可发现性：低**

`=` 和 `-` 作为音量控制不是直觉性的。大多数应用用上下箭头或专门的音量键。

**需要提示：** F1 overlay 已覆盖，但用户需要先发现 F1。

### Save State

**可发现性：中**

F5/F8 是许多模拟器的标准存档快捷键（FCEUX、Mesen）。有模拟器经验的用户会尝试。

**需要提示：** F1 overlay 已覆盖。

### 综合评估

**F1 overlay 是关键。** 所有功能都在 F1 帮助中列出，但用户需要知道按 F1。

**建议：** 首次启动时自动显示 F1 overlay 3-5 秒，然后自动关闭。这会让用户立刻知道有哪些功能。
