# V0.2 EXECUTION PLAN

**基于：** AUDIT_V0.1.md
**日期：** 2026-06-03

---

## 1. Must Have

不做则 v0.2 无法建立在健康基础上。

### 1.1 `let _ =` 错误处理改进

- **价值：** 消除渲染路径中的静默失败，overlay 可见性问题可被诊断
- **风险：** 无。仅替换 `let _ =` 为 `if let Err(e)` 或 `?`
- **工作量：** 0.5 天
- **影响现有稳定性：** 否。行为不变，仅增加错误可见性
- **涉及：** `main.rs:126`, `main.rs:151`

### 1.2 帧率限制改用 vsync

- **价值：** 消除 `thread::sleep` 帧率抖动，画面更流畅，减少 CPU 空转
- **风险：** 低。SDL2 `PresentVSync` 是标准方案，回退到 sleep 无成本
- **工作量：** 0.5 天
- **影响现有稳定性：** 否。渲染管线不变，仅帧同步方式改变
- **涉及：** `main.rs:272-275`（canvas 构建）、`main.rs:529-533`（帧率限制）

### 1.3 输入映射查表

- **价值：** 消除 KeyDown/KeyUp 和 ControllerButtonDown/ButtonUp 的代码重复（各 8 对对称映射），新增按键只需改一处
- **风险：** 无。查表是纯重构，行为完全等价
- **工作量：** 0.5 天
- **影响现有稳定性：** 否。映射逻辑不变，仅数据结构改变
- **涉及：** `main.rs:364-391`, `main.rs:413-456`

## 2. Should Have

提升代码健康度和用户体验，但不做不阻塞 v0.2 发布。

### 2.1 main() 拆分

- **价值：** main() 从 333 行降至 ~100 行，新功能有明确的添加位置
- **风险：** 低。模块边界清晰（audio/input/overlay/rom），拆分路径明确
- **工作量：** 1-2 天
- **影响现有稳定性：** 低。纯结构重构，逻辑不变，但需逐模块验证
- **涉及：** `main.rs` 全文，提取 `audio.rs`, `input.rs`, `overlay.rs`, `rom.rs`

### 2.2 暂停功能 (P 键)

- **价值：** 用户直接感知。TMNT3 战斗中无法暂停是明显体验缺陷
- **风险：** 低。主循环中添加 `paused` 状态，暂停时跳过 `clock_frame()` 即可
- **工作量：** 0.5 天
- **影响现有稳定性：** 否。新增状态变量，不影响现有事件处理和渲染
- **涉及：** `main.rs:351`（事件处理）、`main.rs:494`（模拟推进）

### 2.3 音量控制 (+/- 键)

- **价值：** 用户直接感知。当前音量固定，无法调节
- **风险：** 低。音频回调中乘以音量系数即可
- **工作量：** 0.5 天
- **影响现有稳定性：** 否。新增 `volume` 变量，音频管道不变
- **涉及：** `main.rs:24`（回调）、`main.rs:354`（事件处理）

### 2.4 全屏模式 (F11)

- **价值：** 标准功能。大屏体验提升明显
- **风险：** 低。SDL2 `window.set_fullscreen()` 是标准 API
- **工作量：** 0.5 天
- **影响现有稳定性：** 否。仅添加一个事件分支和窗口状态切换
- **涉及：** `main.rs:354`（事件处理）、`main.rs:266`（窗口创建）

## 3. Nice to Have

有价值但不紧急，可推迟到 v0.3+。

### 3.1 config.toml 配置系统

- **价值：** 后续所有可配置功能的基础（键位、窗口、音频）
- **风险：** 中。引入新依赖（toml 解析），需设计配置结构和默认值回退
- **工作量：** 2-3 天
- **影响现有稳定性：** 中。涉及初始化流程重构，需全面回归测试
- **判定为 Nice to Have 理由：** v0.2 的 Should Have 项（暂停/音量/全屏）均可用硬编码默认值实现，配置系统不是它们的前置依赖。推迟到 v0.3 不阻塞任何功能

### 3.2 Save State

- **价值：** TMNT3 关卡长，存档是刚需
- **风险：** 中。依赖 tetanes-core 的 save state API，序列化格式需验证
- **工作量：** 3-5 天
- **影响现有稳定性：** 低。独立功能，不影响主循环
- **判定为 Nice to Have 理由：** 实现复杂度高，v0.2 应优先完成低风险高收益项

### 3.3 Quick Reset

- **价值：** 死亡后快速重试
- **风险：** 低。调用 `deck.load_rom_path()` 重新加载即可
- **工作量：** 0.5 天
- **影响现有稳定性：** 低
- **判定为 Nice to Have 理由：** 依赖 Save State 的完整度才有最佳体验，单独实现价值有限

### 3.4 FPS Overlay

- **价值：** 开发调试用
- **风险：** 无。复用现有 `draw_text()` 渲染
- **工作量：** 0.5 天
- **影响现有稳定性：** 否
- **判定为 Nice to Have 理由：** 面向开发者，非用户需求

### 3.5 block crate 兼容性

- **价值：** 消除未来 Rust 版本编译风险
- **风险：** 低。跟踪上游更新即可
- **工作量：** 0 天（监控）
- **影响现有稳定性：** 否
- **判定为 Nice to Have 理由：** 当前编译无问题，风险是未来的

  5. 暂停功能                 [Should, 0.5天]
